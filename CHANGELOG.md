## 2026-06-05 补充：`<td>` 内行内格式与块级格式的分界说明

### 变更文件
- `references/html-table-advanced.md`

### 变更内容

#### 单元格内富文本支持列表

**原文**：
```
- ✅ 加粗/斜体/链接
- ✅ 图片（`![](url)`）
- ✅ `<br>` 换行
```

**修改为**：
```
- ✅ 加粗（`**text**` → ne-bold="true"）/ 斜体（`*text*`）/ 链接（`[text](url)`）
- ✅ 彩色加粗（`<font style="color:#DF2A3F;">**text**</font>`）
- ✅ 图片（`![](url)`）
- ✅ `<br>` 换行
```

#### 块级语法说明补充

**原文**：
```
- ❌ **`- item<br/>- item` 在 `<td>` 内不渲染为列表**（Markdown 列表语法在 HTML 表格单元格内不生效）
```

**修改为**：
```
- ❌ **`- item<br/>- item` 在 `<td>` 内不渲染为列表**（Markdown 块级语法在 `<td>` 内不生效；行内语法如 `**bold**` 正常生效）
```

### 根因

语雀 Markdown→Lake 解析器对 `<td>` 内的内容做了行内/块级区分：行内格式（`**bold**`、`*italic*`、`[link]()`、`<font>`）正常解析为原生格式；块级格式（`- item`、`1. item`、`<ul><li>`、引用、代码块）不解析。

### 影响范围

- 周报 HTML 表格中 `**text**` 加粗的使用（已验证可正常工作）
- 未来在 `<td>` 内使用行内格式的场景

## 2026-06-05 表格单元格列表渲染验证

### 变更文件
- `references/html-table-advanced.md`

### 变更内容

#### 1. 修正错误信息：单元格内富文本支持列表

**原文（错误）**：
```
- ✅ 列表（`- ` 无序列表）
- ✅ 代码块
- ✅ 引用
```

**修改为（经验证正确）**：
```
- ✅ Unicode 列表符号（`• item<br/>• item`）
- ✅ 数字列表（`1. item<br/>2. item`）
- ❌ <ul><li> 在 <td> 内不渲染为列表（渲染为 `+ ` 纯文本）
- ❌ - item<br/>- item 在 <td> 内不渲染为列表（Markdown 列表语法在 HTML 表格单元格内不生效）
- ⚠️ 代码块和引用在 <td> 内行为不稳定，建议避免
```

**验证过程**：
1. 先用 `- item<br/>- item` 格式发布到语雀 → 不渲染为列表
2. 改用 `<ul><li>item</li></ul>` 格式 → 渲染为 `+ item` 纯文本，用户截图确认
3. 最终用 `• item<br/>• item`（Unicode 项目符号 U+2022 + br换行） → 正确渲染

**根因**：语雀的 Markdown 解析器在 HTML `<td>` 上下文中不再解析 Markdown 列表语法，也不渲染语义化 HTML 列表元素。只有视觉格式（Unicode 符号 + `<br/>` 换行）能可靠工作。

#### 2. 新增章节：单元格内列表渲染

在"已验证不支持的属性"章节之后，新增独立的"单元格内列表渲染（2026-06 验证）"章节，包含：
- 5种方案的验证结果表格
- 推荐写法的 HTML 代码示例
- `<font>` 内嵌套 `**加粗**` 在 `<td>` 中可正常渲染的说明

### 影响范围
- 周报生成（weekly-report skill）的表格格式
- 任何需要在语雀 HTML 表格单元格内使用列表的场景

上一条变更（2026-06-05 表格单元格列表渲染验证）将 `• item<br/>• item` 标记为"✅ 推荐方案"，但实际上该方案**只有视觉黑点，没有真正的列表缩进对齐**。用户指出后修正。

   - `数字列表` 从 `✅` 改为 `⚠️ 同上，无缩进`
3. **验证结果表**：`• item<br/>• item` 从 `✅ 推荐方案` 改为 `⚠️ 有黑点但无缩进对齐`
4. **新增核心矛盾说明**：需要黄色表头→必须用HTML表格→HTML表格内无法渲染真正的列表

## 2026-06-09 新增：`<colgroup>` API 列宽 + 修正自适应宽度与全宽展示关系

### 变更

1. **新增 `<colgroup>` 列宽比例**：HTML 表格可通过 `<colgroup><col width="N" />` 设定列宽比例，API 写入 body 即生效，无需 Playwright。总和不可超过 750px，format=ymd 读写保留，format=md 读回丢失
2. **修正"互斥"为"前置依赖"**：自适应宽度不是全宽展示的互斥项，而是前置开关——必须先开启自适应宽度，全宽展示按钮才可见。API 创建的表格自适应宽度默认 OFF
3. **明确操作顺序**：colgroup 写比例 → 标宽下拖拽精调 → 开全宽展示（全宽下拖拽可能失效）

### 变更文件

- `SKILL.md`：护栏 #4 拆为 4a(colgroup) + 4b(Playwright)；决策路由新增列宽综合路径
- `html-table-advanced.md`：新增 colgroup 章节（语法/方向指引/硬约束/组合模板/选型决策树）；更新验证表
- `playwright-workarounds.md`：新增列宽控制方案选择流程图；全宽展示代码加状态检查；修正操作顺序；更新概念关系矩阵
- `end-to-end-example.md`：HTML 表格模板加 colgroup；Step 6 改为 colgroup+Playwright 三步流程

### 修正的错误

| 旧描述 | 正确描述 |
|--------|---------|
| `<colgroup>` 不可用（报错 table only allows tr children） | `<colgroup>` 可用，总和 ≤750px |
| 自适应宽度 + 全宽展示互斥 | 自适应宽度是全宽展示的前置条件 |
| 操作顺序：全宽 → 拖拽 | 操作顺序：拖拽 → 全宽 |
